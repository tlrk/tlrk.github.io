interviews to alibaba

近几个月面试了几次阿里巴巴的安卓，做一个小小的总结。
分别是蚂蚁金服、UC、优酷的来疯直播，面试前做了一些准备，但是结果不是很好，下面写出几点感受以及需要在后面去学习的地方

感受

  • 众所周知，阿里巴巴对 java 的要求比较高，注重基础，包括java的内存模型，java虚拟机的原理，多线程和并发，jdk中的一些集合类等，面经中都有提到，但是面试官对深度要求比较高，所以还是踏实的理解背后的原理，融会贯通比较重要。
  • android 的基础知识，包括四大组件，尤其是 activity 和 service 的知识，界面的绘画机制，自定义控件，消息机制的深入理解，hander的理解，结合多线程，进程间的通信，AIDL, android 应用的优化，包括内存泄露，OOM，界面过度绘制，ANR，会问道对应的工具以及相关的只是。三次面试都提到了在工作经验中对多线程并发的理解，这点特别重要。总的感觉是对知识不能一知半解，面试官会深挖直到问倒你为止。这些知识需要自己慢慢积累，并且在平常工作中尽量不要停留在表面，深入问题本质，知道自己没有问题为止，所以需要费很大的功夫。
  • 算法相关的知识，平常工作中对算法使用并不多，并不能代表不用去学习这些知识。我面试后的时候基本上只复习了六种常见的排序（包括时间空间复杂度，稳定性，实现代码），几种常见的查找算法，二叉树相关的知识，字符串的相关操作。从面试的经验上看，如果你没时间去准备其他的知识，上面的那些知识复习到位也就能够应对90%以上，毕竟不是专业搞算法。但是如果有兴趣还是多研究下，不会有坏处。
  • 设计模式的相关知识。因为比较虚，所以自己也没准备多少，恰巧在来疯直播中有让我写单例和观察者模式，我没有做得很好，当时的感觉时，这些知识要是平常基本上网上一搜就都有了，所以并没有多少必要让自己能够记下来。还是自己准备不是很充分吧，想尝试一些架构方面的东西，基本的23种设计模式应该是要滚瓜烂熟的，还得好好准备。另外，处理理论知识，面试官会问道关于在代码中对设计模式的运用，这些只能靠自己平常的写代码经验和总结了。

后面要去学习的地方

  • 服务，包括服务的使用，不同的启动服务方式，背后的原理，远程服务等。
  • activity 的基本知识
  • 23种常用的设计模式以及对应的代码实现
  • 深入对算法的理解
  • 在 android 的网络操作，包括例如 OkHttp 的封装原理
  • android 的优化知识
  • 界面的相关知识，包括自定义控件，消息机制，SurfaceView 等
  • java 基础知识，包括jdk的源码解读，jvm的原理，内存模型，一些关键字的理解，涉及到锁方面的问题
  • 多线程和进程间通信

how to solve it

生活中解决问题的思路

回忆下生活中我们解决问题的思路，首先问题产生了，问题有一个上下文，就是问题产生的环境，如果抽象到题目，就是题目中的前提条件。面对问题开始分析，分析问题的细节和一些边界，利用联想、类比或者归纳等不同的解题方法，采取一定的措施尝试解决问题，最后找到最优解或者认为的最优解，验证解题结果是否在可以接受的范围内，然后总结成为脑海中的一种经验，结束。

学校给我们的教育

拿数学的定理或者一个推理来说，首先是有了结论，然后根据课本上的介绍，按照一步一步的推导证明，最终得到正确的答案，老师告诉我们应该记住这种推导方式，然后下次遇到相同的问题可以按照类似的方法同样推导下去。其实很疑惑的一点，为什么书中讲到的论证步骤应该是那样，为什么这个定理的作者能够想到那样的推论而我自己不能想到，为什么某些步骤非常巧妙的解决了问题但是我还不知道怎么突然冒出这样一步……拿着结论去找原因，模仿解法，没有自己的思考，不会有更深刻的印象，对于一个问题仅仅是找不到解而已，但是对于自己的思维受到了限制，确实非常严重的一件事。

解决问题的思路应该是怎么样

首先是明白问题的已知量和未知量，边界条件，问自己几个问题，这个问题我知道什么，我需要解决什么问题。我们就是一些解题思路的问题，包括根据经验的联想，对问题的一般化或者具体化，这个问题本质上是什么问题，对于这个问题，之前有没有好的解法，能不能自己想出不同的解法，解出问题后，总结一下问题，如果有兴趣，还可以探究下为什么会出现这样的问题，有时候寻找问题出现的原因比寻找问题的解更有意义，层次更加深。

我学到了什么

按照认识事物的本质去认识问题，顺序一定不要颠倒，尤其是很多技术书籍，一出来就是讲解一通源码，实现方式，没有交代前面一些有意义或者有意思的故事，就好像直接给出了结论，这样的思路是不对了，对于技术的学习，还是要尊重认识事物的过程和原则。一个新技术的出现，势必是为了解决对应的问题，那么首先是之前相关的技术没有满足需要，出现了一些矛盾的东西，因此为了解决这个矛盾，新的技术才出现。那么矛盾或者问题是什么，为什么之前的技术没有很好解决那个问题，为什么新技术能够解决这个问题，或者新技术是怎样解决矛盾的，解决的怎么样，有什么利弊，在技术层面，没有百分之百的完美，基本都是一种权衡，本质上是时间和空间的权衡，因此新技术必然有利弊，如何在熟练新技术的技术上，避开弊端，利用优势，才是掌握新技术的精髓所在。
